# Asset conventions

## Goals
- stable public paths
- predictable image loading
- fewer broken references on GitHub Pages

## Naming rules
- lowercase only
- hyphen-separated slugs
- no spaces
- no inconsistent team naming

## Suggested patterns
- `assets/img/logos/optic-texas.png`
- `assets/img/players/optic-texas/shotzzy.webp`
- `assets/img/team-stats/optic-texas/overall.png`

## Recommendation
Keep the private source assets untouched if needed, and create a normalized public asset copy for the GitHub project.
